# ☁️ 云端部署注意事项与运维指南

## 1. 服务器重启后的行为 (Restart Policy)

当您的云服务器关机重启后（或者崩溃恢复后），由于我们在 `docker-compose.yml` 中设置了：

```yaml
restart: unless-stopped
```

这意味着：
*   ✅ **无需重新部署**：只要 Docker 服务（Docker Daemon）随系统启动，您的游戏服务容器就会**自动启动**。
*   **例外情况**：如果您在关机前手动使用了 `docker compose down` 或 `docker stop` 停止了服务，重启后它们**不会**自动启动。此时需要手动运行 `docker compose up -d`。

**如何确保 Docker 开机自启?**
在服务器上执行一次：
```bash
sudo systemctl enable docker
```

## 2. 长期运行 (不关机) 的注意事项

如果您的服务器长期不关机运行，需要关注以下几点以保证服务稳定性：

### A. 磁盘空间管理 💾
Docker 容器长期运行会产生日志和临时文件。
*   **日志积压**: 默认情况下 Docker 可能会保留大量日志。建议配置日志轮转 (Log Rotation)。
    *   *检查命令*: `docker system df` 查看空间占用。
    *   *清理命令*: `docker system prune -f` (慎用，会删除未使用的镜像和缓存)。

### B. 内存泄漏监控 🧠
虽然 ROS 2 节点比较稳定，但长期运行可能存在内存缓慢增长的情况。
*   使用 `htop` 或 `docker stats` 偶尔查看内存占用。
*   如果发现内存占用过高，可以设置定时任务 (Cron Job) 每周重启一次容器：
    ```bash
    # 每周一凌晨 3 点重启服务
    0 3 * * 1 cd /path/to/project && docker compose restart
    ```

### C. 安全与更新 🛡️
*   **系统更新**: 即使服务不重启，操作系统层面的安全补丁仍需安装。
    ```bash
    sudo apt update && sudo apt upgrade -y
    ```
*   **端口安全**: 确保腾讯云/阿里云的安全组规则只开放了必要的端口 (80, 9090)，关闭 SSH (22) 的全网访问（建议限制为特定 IP 或使用密钥登录）。

### D. 费用控制 💰
*   如果是**按量计费**实例，长期开机即便没有流量也会扣费（CPU/内存/公网IP租赁费）。
*   如果是**包年包月**，则无需担心。

## 3. 重新部署的场景
您只需要在以下情况**重新部署** (执行 CD 流水线或手动 `docker compose up --build`)：
1.  **代码更新**: 修改了前端 HTML/JS 或后端 Python 逻辑。
2.  **配置变更**: 修改了 `docker-compose.yml` (例如端口映射、环境变量)。
3.  **环境升级**: 更新了基础镜像或安装了新的系统库。
