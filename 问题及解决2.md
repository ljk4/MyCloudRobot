# 问题及解决记录 2 (Docker 优化与权限修复)

本文档记录了在 Docker 镜像构建优化及非 root 用户权限配置过程中遇到的问题及解决方案。

## 1. Nginx 启动失败：Permission Denied (HTTP 502)

### 问题描述
容器启动后，访问 Web 页面出现 `502 Bad Gateway`。查看容器日志发现 Nginx 无法启动，报错权限不足。
**错误日志：**
```
nginx: [alert] could not open error log file: open() "/var/log/nginx/error.log" failed (13: Permission denied)
2026/02/25 13:28:44 [emerg] 28#28: mkdir() "/var/lib/nginx/body" failed (13: Permission denied)
```

### 原因分析
出于安全考虑，我们将容器后的运行用户切换为了非 root 用户（`robotuser`）。
默认的 Nginx 配置试图将日志写入 `/var/log/nginx`，将临时文件写入 `/var/lib/nginx`。这些目录通​​常只有 root 用户有写权限，导致普通用户启动 Nginx 失败。

### 解决方案
修改 `docker/nginx.conf`，将所有需要写入的路径（日志、PID 文件、临时文件）都指向 `/tmp/` 目录，因为 `/tmp/` 目录对所有用户都是可写的。

**修改后的 `nginx.conf` 片段：**
```nginx
# 错误日志改到 /tmp
error_log /tmp/error.log warn;
pid /tmp/nginx.pid;

http {
    # 访问日志改到 /tmp
    access_log /tmp/access.log main;

    # 关键：设置非 root 用户可写的临时目录
    client_body_temp_path /tmp/client_body;
    proxy_temp_path /tmp/proxy;
    fastcgi_temp_path /tmp/fastcgi;
    uwsgi_temp_path /tmp/uwsgi;
    scgi_temp_path /tmp/scgi;
    
    # ... 其他配置
}
```

## 2. 构建速度过慢

### 问题描述
每次修改代码后运行 `docker compose build`，都需要重新下载大量 apt 依赖和 ROS 包，加上原本使用的是官方源，构建过程非常耗时（超过 10 分钟）。

### 解决方案
采取了 **"基础镜像 + 应用镜像"** 的分层策略，并配置了**国内镜像源**。

1.  **创建基础镜像 (`Dockerfile.base`)**：
    *   包含所有耗时的系统依赖（ROS 核心包、Nginx、编译工具）。
    *   配置了清华大学（Tsinghua）的 apt 源和 ROS 2 源。
    *   预先构建好并打标签：`docker build -t ljk4/cloud-robot-viz-base:latest -f docker/Dockerfile.base .`

2.  **优化应用 Dockerfile**：
    *   `FROM ljk4/cloud-robot-viz-base:latest` 直接使用预构建好的环境。
    *   构建过程只需要 `COPY` 源代码并运行 `colcon build`，可以在几秒钟内完成。

## 3. GPG 签名错误 (NO_PUBKEY)

### 问题描述
在 `Dockerfile` 中更换为清华源后，执行 `apt-get update` 报错，提示没有公钥：
```
The following signatures couldn't be verified because the public key is not available: NO_PUBKEY ...
```

### 解决方案
在添加源之前，必须先下载并保存 ROS 的 GPG 密钥。

**Dockerfile 中的修复命令：**
```dockerfile
RUN apt-get update && apt-get install -y curl gnupg2 \
    && curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key -o /usr/share/keyrings/ros-archive-keyring.gpg \
    && echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] https://mirrors.tuna.tsinghua.edu.cn/ros2/ubuntu jammy main" > /etc/apt/sources.list.d/ros2-latest.list
```

## 4. 静态文件路径错误 (404 Not Found)

### 问题描述
Nginx 启动成功，但访问页面提示 404，或者看到的不是预期的页面内容。

### 原因分析
1.  **路径不匹配**：Nginx 默认配置指向 `/usr/share/nginx/html`，但我们将 Web 文件复制到了 `/home/robotuser/web`。
2.  **User 切换**：切换到非 root 用户后，部分目录变得不可读。

### 解决方案
1.  在 `nginx.conf` 中修改 `root` 指令：
    ```nginx
    server {
        location / {
            root /home/robotuser/web;  # 指向正确的位置
            index index.html;
        }
    }
    ```
2.  在 `docker-compose.yml` 中同步修改挂载卷：
    ```yaml
    volumes:
      - ./web:/home/robotuser/web
    ```
3.  在 Dockerfile 中确保文件权限正确：
    ```dockerfile
    RUN chmod -R 755 /home/robotuser/web
    ```
